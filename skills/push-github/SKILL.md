# Push to GitHub

Push my code changes to GitHub with a clear commit message.

## Steps

1. Check the current git status to see what files changed
2. Stage all modified files
3. Create a descriptive commit message based on the changes
4. Commit the staged changes
5. Push to the current branch
6. Show a summary of what was pushed

## Guidelines

- Commit message should describe WHAT changed and WHY
- Be concise but specific
- If there are errors, explain what went wrong and why
- Show the branch name and number of commits in the summary

## Example Output

Pushed to GitHub
├── Branch: main
├── Commits: 1
├── Message: "feat: add user authentication"
└── Status: ✓ Success
